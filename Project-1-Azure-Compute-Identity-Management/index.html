Key Concepts
RBAC vs. Access Policies
Key Vault supports two permission models. Azure RBAC (recommended) integrates with Entra ID using standard role assignments. The legacy access policies model is being retired — always choose RBAC for new deployments.
Groups Over Individuals
Assigning roles to Entra ID groups means onboarding and offboarding is a group membership change — not a series of role reassignments across every resource. One change propagates everywhere.
Static Private IP
Dynamic IPs can change on restart. Set static when NSG rules, DNS entries, or monitoring configs reference the IP directly — any change to a dynamic IP breaks those dependencies silently.
Disk Init in Windows
Azure data disks arrive uninitialized. Azure handles provisioning the disk to the VM; Windows Disk Management handles the filesystem — identical to initializing a new SAN LUN on any Windows Server.
Policy Initiatives vs. Policies
An initiative bundles multiple policies into a single assignable object. Update the initiative once — all assignments automatically inherit the change without individual reassignment.
Budget Alerts ≠ Spending Limits
Alerts notify you — they don't stop charges. Azure won't automatically shut off resources at the budget threshold. Use alerts as triggers for manual review or automated runbook actions.
