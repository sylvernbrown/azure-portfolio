Walkthrough
Step 01
Enable VM Insights
  1. Navigate to vm-win-lab01 → Monitoring → Insights → Enable
  2. Create or select a Log Analytics Workspace
  3. Enable — Azure installs the monitoring agent automatically
VM Insights enabled — monitoring agent deployed to vm-win-lab01
Log Analytics workspace created — logs and metrics will flow here
Step 02
Configure a CPU Alert
  1. Navigate to vm-win-lab01 → Monitoring → Alerts → + Create → Alert rule
  2. Signal: Percentage CPU
  3. Condition: greater than 80% for 5 minutes
  4. Create an Action Group → add your email
  5. Name the alert rule and save
Alert condition — CPU above 80% triggers email notification
Action group configured with email notification
Step 03
Create a Recovery Services Vault and Configure Backup
  1. Navigate to Recovery Services vaults → + Create
  2. Name: rsv-lab-01, same region as the VM
  3. Inside the vault: Backup → Azure Virtual Machine
  4. Select vm-win-lab01 → Enable backup
  5. Policy: Daily, retention 7 days minimum
  6. Run Backup Now to verify it completes
Recovery Services Vault created — rsv-lab-01
Daily backup policy configured — 7-day retention
Backup job completed successfully
Step 04
Configure Azure Site Recovery

Site Recovery replicates the VM to a secondary region. If the primary region goes down, you can fail over with minimal downtime.

  1. Inside the Recovery Services Vault → Site Recovery → Azure virtual machines → Enable replication
  2. Source: current region, target: secondary region (e.g., East US)
  3. Select vm-win-lab01
  4. Review and enable replication — initial sync takes time
Site Recovery replication enabled — VM being replicated to secondary region
Replication status — initial sync in progress
Step 05
Write KQL Queries in Log Analytics
  1. Navigate to your Log Analytics Workspace → Logs
  2. Run queries to analyze VM performance:
// Average CPU over last hour
Perf
| where ObjectName == "Processor" and CounterName == "% Processor Time"
| summarize avg(CounterValue) by bin(TimeGenerated, 5m)
| render timechart
KQL query — CPU utilization over time rendered as a time chart
Log Analytics workspace — performance data flowing in from vm-win-lab01
Key Concepts
Alerts vs. Insights
VM Insights gives you visibility — dashboards, maps, performance charts. Alerts act on that data — notify you or trigger automation when thresholds are crossed.
Backup vs. Site Recovery
Backup protects against data loss — restore a file or a VM from a point in time. Site Recovery protects against regional outages — fail over a live VM to another region.
KQL Basics
Kusto Query Language — pipe-based syntax similar to SQL. Table | where | summarize | render. Used across Azure Monitor, Log Analytics, Sentinel, and Defender.
Recovery Services Vault
A single vault handles both Backup and Site Recovery. Stores backup data encrypted at rest. Policies define retention schedule — production typically 30-90 days.
